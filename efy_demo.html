What's the logic behind it
  • Minimal efficient code
  • The user is in control, everything is modular
  • Developers & designers save a ton of time, and only need to combine patterns & choose defaults, but the rest doesn't need to be rebuilt each time like with traditional frameworks
  • It's accessible (not fully yet)
  • Sound effects - for a more cinematic & immersive experience
  • Touchscreen friendly & convergent - which is why hovering or using a cursor are not requirements for interracting with the ui, something that is usually not fully taken into consideration by devs / frameworks. Here it is!
  • Transparency friendly: no popups or multilayered elements when possible. Almost close to 0 projects support this currently, so now you can, with a fallback to solid / image backgrounds as well.
  • Things are easily searchable & open on demand when possible to reduce physical space and help you focus
Can I use it on my own projects?

Yes, but please donate if you want to support the project and can afford it, that would help a lot! Read the license for more details.

efy_theme
  • It sets the theme, the default value is light_light
  • Options: light_light light_grey light_sepia light_trans dark_dark dark_nord dark_black dark_trans

    • Structure

  • 1st part: can be light or dark to indicate the general mode
  • 2nd part: indicates the specific type of light / dark mode: light grey sepia trans dark nord black
  • trans = Transparent - it makes the background invisible so that you can see behind the window (if your web engine, browser or operating system supports it) or the image you choose from the sidebar menu

Examples

Specific theme:

[efy_theme=light_sepia] { your css }

All light themes:

[efy_theme*=light] { your css }

Not this specific theme:

:not([efy_theme=light_grey]) { your css }

Remove selector if any light theme is on, in JS

e$('[efy_theme*=light] #your_selector').remove()
Accessibility

efy_outline - adds an outline around elements for accessibility with the color set by --efy_text . On touchscreen it becomes invisible (since without a mouse there's no point in making it visible)

efy_cursor_on - adds a custom cursor, which takes colours from --efy_color & --efy_color2
